# 競合ベンチマーク：Lai（l-ai.jp）→ Keiro機能バックログ反映

作成: 2026-07-20 ／ 出典: https://l-ai.jp/ （Meta広告経由で確認）

## 競合の概要

**Lai** = LINE公式アカウント運用のAIエージェントSaaS。チャットで依頼→AIが設定の「下書き」を管理画面に反映→人間が承認して実行（個別チャットと即時配信はAI送信禁止）。訴求は「作業時間を最大90%削減」。

料金: Starter ¥9,800（5,000通/API無し）／ Standard ¥24,800（30,000通/API有り）／ Pro ¥59,800（100,000通）。7日間無料トライアル。

## 変更点（反映にあたって必要な追加・変更）

### P1: Laiの中核訴求への対抗（効果大）

1. **AI下書きエージェントの範囲拡大** — richmenuChatで実績のある「チャットで依頼→下書き→人間が承認」方式を、一斉配信文面・ステップ配信シナリオ・自動応答の作成にも拡大する。Laiと同じく「個別チャットと即時配信はAI禁止」の安全ガードを踏襲・明文化する。基盤（aisetup.jsのcallClaude/callGemini）は流用可能。
2. **AI分析チャット** — 「URLをクリックした人の属性を分析して」等の自然言語依頼に、自テナントのクリック・友だち・配信データを集計して回答する機能。Laiの時間削減事例2（4時間10分→数秒）への直接対抗。
3. **ステップ配信の条件分岐** — steps.jsにタグ・流入経路・フォーム回答による分岐を追加（現状は直列のみ）。Phase1の「自己申告→タグ→分岐」実装と統合して設計する。
4. **汎用ワークフロー** — 「トリガー（タグ付与/経過日数/フォーム送信）→条件→アクション（配信/タグ/クーポン）」のルールエンジン。誕生日・リマインダー等の既存固定機能を将来的にこのエンジン上へ統合。

### P2: 見せ方・運用の強化（効果中）

5. **ファネルUI** — クリック→友だち追加→フォーム到達→送信完了の段階可視化（Laiの「1,020→486→213」型の表示）。データは既にあるのでUI追加が主。
6. **タグセグメント** — タグの組合せ（AND/OR）を保存し、一斉配信・ステップ配信のオーディエンスとして再利用できるようにする。
7. **店舗内権限ロール** — オーナー／運用担当の2ロール追加（現状は店舗=1アカウント）。

### P3: 連携系（上位プランの差別化材料）

8. **公開API** — 友だち・タグの読み書きAPI＋APIキー発行。Laiに合わせ「上位プラン限定」にすると料金表の差別化材料になる。
9. **Webhook転送** — 受信したLINEイベントを指定URLへそのまま転送。実装コスト小。

## 営業・料金面の示唆（機能外だが記録）

- Lai Starter ¥9,800とKeiroプロ¥9,800が同額。ただし**広告CVポストバック（Meta/TikTok）と認証画面なし計測はKeiroだけ**。営業資料では「広告の費用対効果まで見えるのはKeiro」を明記して差別化する。
- Laiの「AIは下書きまで、承認は人間」という安全性訴求は顧客受けが良い。Keiroも同方針である旨をLP・ガイドに明文化する価値あり。
- Laiは7日間無料（Keiroは30日）。無料期間の長さはKeiroの優位点として訴求可。
